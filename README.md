# Zepto ecommerce-inventory-sql-analysis
SQL-based data analytics project focused on inventory management, pricing analysis, stock availability, and business insights using an e-commerce dataset.
🛒 E-Commerce Inventory SQL Analysis
📌 Project Overview

In this project, I analyzed an e-commerce inventory dataset using SQL to gain insights into product performance, pricing strategies, stock availability, and inventory management. The project simulates real-world data analyst workflows, from data exploration and cleaning to generating business-driven insights that support decision-making.

🎯 Project Objectives
- Design and manage a structured inventory database.
- Perform exploratory data analysis (EDA) on product and inventory data.
- Clean and transform raw data to improve quality and consistency.
- Generate actionable business insights using SQL queries.
- Analyze pricing, discounts, inventory levels, and category performance.

📁 Dataset Overview
- The dataset contains inventory information for products across multiple categories in an e-commerce environment.
- Each record represents a unique product SKU and includes information related to pricing, inventory availability, product categories, discounts, and package details.

Dataset Fields
1. SKU ID – Unique identifier for each product
2. Product Name – Name of the product
3. Category – Product category
4. MRP – Maximum Retail Price
5. Discount Percentage – Percentage discount applied
6. Discounted Selling Price – Final selling price after discount
7. Available Quantity – Inventory available for sale
8. Weight (grams) – Product weight
9. Out of Stock Status – Product availability indicator
10. Quantity – Number of units per package

🔧 Project Workflow
1. Database Creation
Created a SQL database and designed a structured table using appropriate data types and constraints to store inventory data efficiently.

2. Data Import and Preparation 
- Imported the dataset into PostgreSQL.
- Resolved data formatting and encoding issues.
- Validated imported records to ensure data accuracy.

3. Exploratory Data Analysis (EDA)
Performed exploratory analysis to:
- Determine the total number of records.
- Identify unique product categories.
- Analyze inventory availability.
- Detect duplicate product entries and SKU variations.
- Examine overall dataset quality.

4. Data Cleaning
Conducted data cleaning processes including:
- Removing invalid records.
- Handling missing or inconsistent values.
- Standardizing pricing values for analysis.
- Improving data consistency and reliability.

5. Business Analysis
Used SQL queries to generate insights such as:
- Top products with the highest discounts.
- High-value products currently out of stock.
- Revenue potential across product categories.
- Category-wise discount analysis.
- Inventory distribution across product groups.
- Value-for-money products based on pricing and weight.
- Product segmentation based on package size and weight.

📊 Key Insights
1. Identified product categories with the strongest revenue potential.
2. Analyzed discount patterns across categories.
3. Evaluated inventory availability and stock-out risks.
4. Measured category-level inventory distribution.
5. Highlighted products offering the best customer value.
6. Generated insights to support pricing and inventory decisions.
